# TinyAya Stage 2 — Public Release Plan (v0.3, audio-only)

Goal: make **everything** public — models, all checkpoints, datasets, code, and an
honest writeup — so the project is a fully transparent, reproducible account of a
low-resource Turkish↔Hindi speech-to-speech translation attempt, including its
real data limitations. Honesty *is* the deliverable.

> Status: drafted 2026-06-30, ahead of the v0.3 full-1.24M audio run. Tick items
> as released. The v0.3 model card is held until the run produces weights.

---

## 1. The honest narrative (the spine of the release)

Three runs, one corrected story — tell it straight:

| Version | Data (actual) | Outcome | Honest note |
|---|---|---|---|
| **v0.1** | synthetic `tr-hi-mimi-encoded` (1.18M) | audio learned; **text stream did not** | We now know *why*: the corpus has **no text alignments** → the Moshi inner-monologue can't be supervised. |
| **v0.2** | `fleurs-tr-hi-mimi-encoded` (FLEURS, ~8.3k) | overfit (val bottomed @ step 1000) | Trained on the **wrong dataset** — the launcher's `HF_DATASET` defaulted to the `fleurs-` sibling. Disclosed on the card. |
| **v0.3** | synthetic `tr-hi-mimi-encoded` (1.24M) | **audio-only**, regularization-swept | The right data + the v0.2→v0.3 audio fixes (regularization, deep-codebook). Text is structurally absent (no alignments), so trained audio-only. |

**Key public findings (do not bury these):**
1. **The synthetic corpus ships no per-sample text alignments** (verified: a batch
   tarball has 270k `.pt`, 0 `_src/_tgt.json`). So the inner-monologue/text stream
   is untrainable on this data — v0.3 is honestly **audio-only S2ST** (CB0 from the
   Cohere2+LoRA backbone, CB1–7 from the frozen Moshi depth decoder, Mimi codes).
2. **Dataset attribution was wrong on v0.1/v0.2 cards** and is now corrected
   (v0.1 = synthetic but mislabeled "FLEURS"; v0.2 = actually FLEURS).
3. **Regularization sweep winner** (audio-only `val/composite` = `val_audio`):
   `lora_r=8, lora_dropout=0.15, weight_decay=0.10, lr_lora≈2.8e-4` — the
   most-regularized + higher-LR combo, i.e. the anti-overfit direction.

## 3. Evaluation (must run before the v0.3 card is final)
Audio-only, so report what the model actually does — don't over-claim translation:
- [ ] **Per-codebook accuracy** (teacher-forced) on held-out val — CB0 vs CB1–7.
- [ ] **ASR-BLEU** (Whisper transcribe generated target audio → BLEU vs reference) —
      expect low; report honestly (the model learns audio codes; sentence-level
      translation likely hasn't emerged, mirroring v0.1).
- [ ] **Audio naturalness** (DNSMOS/UTMOS) on decoded Mimi.
- [ ] State protocol: checkpoint used (`best_by_val`), ASR backend+version, decoding,
      seed, #samples, text normalization.
- Note the eval harness delay-indexing was verified correct (CB1–7≈0 is real, not a
      read bug).
