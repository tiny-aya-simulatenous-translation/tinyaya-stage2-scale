# v0.3 Training — Implementation Plan (PR #10)

**Target PR:** #10. **Operationalizes** the Phase A–E strategy from
[`docs/next-v0.3-training-plan.md`](next-v0.3-training-plan.md) (PR #9) into
concrete, file:function-level code work. Same principles (phased, table-driven,
EXA-grounded) — but this is the **engineering blueprint** + execution order, not
the rationale (see #9 / its References for the *why* and citations).

**Goal:** ship `tr-hi-s2st-v0.3` where **validation improves over training**, deep
codebooks learn, and both streams stay healthy — by *implementing* the levers #9
identified.

## ⏱️ Revised execution order (owner decision)

**Phase B (weight averaging) is deferred to the END**, gated on evals — *not* a
"step-0 do-it-now" item. Rationale: averaging only helps if you can **measure** that
it improves a score, which requires evals; and the v0.2 checkpoints (few, no evals
yet) aren't worth blind-averaging. So the order is:

1. **Basic evals on v0.2** (establish a baseline score — ASR-BLEU at minimum).
2. **Implement + train v0.3** = Phase **A** (now) → **C** → **D** → **E**.
3. **Basic evals on v0.3.**
4. **Phase B (averaging) last** — average the final 2–3 v0.3 checkpoints (and/or v0.2)
   and re-eval; keep it only if it **improves the eval score**.

So `scripts/average_checkpoints.py` is **not built now**; it's a late-stage,
eval-gated refinement. The **immediate work is Phase A**.

## Per-phase implementation detail

### Phase A — Regularization (config + 2 edits + early-stop)
- **`configs/tpu/stage2_tpu_v6e_v3.yaml`** (new): `lora.r=16, alpha=32, lora_dropout=0.08`;
  `optim.lr_lora=1e-4`; `train.weight_decay=0.05, grad_clip=1.0`;
  `loss.label_smoothing=0.1, text_padding_weight=0.01`; scheduler `min_lr_ratio=0`,
  `warmup_steps≈8%`.
- **`lora_setup.apply_lora`**: add `lora_dropout: float = 0.0` → `LoraConfig(lora_dropout=...)`; plumb from config.
- **`translation_loss.compute_hierarchical_translation_loss`**: add `label_smoothing: float = 0.0`
  → pass to every `F.cross_entropy(...)` (works with `reduction="none"`).
- **Early stopping** (`train_hierarchical` val block): track `best_composite`, `patience_left`;
  if composite val worsens for `early_stop_patience` (≈4) cycles → break. Reuse `best_by_val` save.

### Phase B — Weight averaging (DEFERRED to the end, eval-gated)
> **Not built now.** Moved to the final, eval-gated step (see *Revised execution
> order* above). Only worth it once basic evals exist so we can *measure* whether
> averaging improves the score. The v0.2 checkpoints (few, un-evaluated) aren't worth
> blind-averaging.
- **`scripts/average_checkpoints.py`** (CPU, torch.load), built **after** v0.3 trains:
  given a list of checkpoint dirs (each = `*.pt` components + `peft_adapter/adapter_model.safetensors`),
  produce one **averaged** checkpoint with the same layout. Modes: `uniform`/SWA, `ema`-decay,
  `topk`-by-eval-score. Averages **both** the `.pt` tensors and the adapter safetensors.
- **When:** average the final **2–3 v0.3 checkpoints** (and/or v0.2) → **re-eval** → keep the
  averaged model only if it beats the best single checkpoint on the eval metric.
- **Optional in-training EMA** (`src/training/ema.py`, flag-gated): shadow copy of trainable params.
  ⚠️ Under **FSDPv2/SPMD** params are sharded → high implementation cost. **Defer**; post-hoc
  averaging captures most of the benefit at far lower risk.

### Phase C — Deep-codebook learning (loss hook + freeze lever)
- **`translation_loss`**: replace `audio_loss = per_codebook.mean()` with
  `audio_loss = (per_codebook * cb_weights).sum() / cb_weights.sum().clamp(min=1)`, where
  `cb_weights: [num_cb]` is **passed in** (keeps the loss pure; XLA-traces fine — same unrolled loop).
- **Schedule helper** (train loop): `cb_weights(step)` = per-codebook multipliers (start *gentle*, e.g.
  `[1,1,1.5,1.5,2,2,2,2]`, not the report's aggressive `[1..128]`) **× an unmask mask** that activates
  codebooks `0..K(step)-1`, where `K` ramps `K0→num_cb` over `progressive_unmask_fraction × total_steps`.
- **Depth lever** (`freeze_depth_internals` + `get_param_groups`): `depth_unfreeze_blocks`
  unfreezes the last N depth transformer blocks at **setup** (not mid-run — avoids FSDP
  param-group surgery) into a low-LR `depth_blocks` group (`lr_depth_blocks`, default
  `lr_depth×0.1`). The Phase-C2 unmask gates their gradients, so they stage in once their
  codebooks unmask. Opt-in (default 0); rely on unmask + A first.
  - ⚠️ **TPU-smoke finding (2026-06-28):** `xla_grad_checkpoint` wraps `.layers` in a
    `_ScannedLayerStack`, so block params are named `…layers.layers_list.<i>.layer.*` and
    module-walk over `.layers` sees 0M. Selection is therefore **name-based**
    (`param_classify.depth_block_layer_index`, regex handles `layers_list`) off
    `depth_decoder.named_parameters()`, with an `assert uf>0` guard. Verified live:
    unfreeze 206M, `depth_blocks` group 205.5M @ lr 1e-5.

### Phase D — Multitask balance
- **`run_validation`**: also return `val/composite = 0.4·val_text + 0.6·val_audio`; use it for
  `best_by_val` + early-stop. Per-stream + per-codebook already logged.
- **`text_weight` curriculum** (train loop): anneal `0.5→0.3` over first ~25% (simple), **or** adaptive
  gating: if a stream's `L_val/L_train > 1.10–1.15`, nudge its weight ×1.05–1.15. Start with the simple
  curriculum; gating is a follow-up.

### Phase E — Sweep redesign, data, run
- **`sweeps/sweep_stage2_v3.yaml`**: sweep **regularization** (`lora_r∈{8,16,32}`, `lora_dropout`,
  `weight_decay`, `lr_lora`) on a **longer proxy** (~2–3k steps), metric = **best early-stopped composite
  val** (not 600-step train audio loss). Extend `promote_sweep_winner.py` PARAM_MAP for the new knobs.
- **SpecAugment-on-codes** (`src/data`, flag-gated): classic SpecAugment is on spectrograms; on discrete
  Mimi codes adapt to **random time-masking** (mask spans of code frames → `SILENCE_TOKEN`) +
  **codebook dropout**. Apply in the collator/dataset; shape-preserving.
- **Run order:** smoke (all levers fire) → regularization sweep → v0.3 run (early-stopped) →
  `average_checkpoints.py` on the run → **release the averaged weights** as `tr-hi-s2st-v0.3`.

## Run / experiment sequence (owner-revised)

1. **Eval v0.2** — basic evals (ASR-BLEU min) on the existing model → **baseline score**.
2. **Implement Phase A (now)** → C → D → E. **Smoke** on the v3 config: confirm dropout, label-smoothing, cb-unmask schedule, and early-stop all fire.
3. **Regularization sweep** (`sweep_stage2_v3.yaml`) → `promote_sweep_winner` → `v6e_v3.yaml`.
4. **v0.3 run**, early-stopped on composite val → **eval v0.3**.
5. **Phase B (last):** average the final 2–3 v0.3 checkpoints (and/or v0.2) → **re-eval** → release the averaged model **only if it beats the best single** on the eval metric.
